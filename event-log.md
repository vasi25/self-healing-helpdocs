# Event Log

*Written by Luiza Gircoveanu · April 29, 2026*

## Introduction

Event Log is a section in the Settings menu used to examine change history. Using different filters, it acts like a real-time console for your in-app activity, such as creation, deletion, or updates to Audits, Journeys, Consent Categories, and more.

## Main filters

The Event Log can be found in the left navigation bar, under Settings. It has 5 different filters that can be used to see specific data history:

- **Item Type** — Account, Audit, Journey, Consent Category, Folder, Sub-Folder, Email Inbox, Saved Report
- **User** — Select a user from the list
- **Event Type** — Created, Edited, Deleted, Run Requested, Overages Enabled, Overages Disabled
- **Item ID** — Fill in a specific ID you would like to check on
- **Timestamp** — Choose a period of time to see the events

You can fill in either all filters or just select ones, based on your needs. Once you hit Submit, the log updates.

The report shows fields such as:

- Item type
- User ID
- Username
- Event Type
- Item ID
- Item Name
- Timestamp

## Common use cases

- See when an Audit was created, ran, or edited.
- Identify who deleted a rule and when it was deleted.
- See who enabled Allow Overages in your account and when.

## Conclusion

The Event Log can be used to see history changes based on filters such as Audits, Journeys, or Consent Categories. It's the best resource in the ObservePoint platform to identify who made changes in configuration. If you don't find the information you need in the event log, contact ObservePoint support and they'll help you find it.
